# PRD: Realtime Caption Studio

## 1. 문제 정의

### 현재 상황
속기사는 실시간 행사(컨퍼런스, 강연, 방송, 국회 회의 등)에서 발화를 즉시 텍스트로 변환하는 전문가다. 현재 속기 업계의 문제점:

- **교대 시스템 부재**: 장시간 행사에서 2~4명의 속기사가 교대하지만, 교대 과정이 수동적이고 끊김이 발생
- **도구 파편화**: 입력 도구, 자막 출력 도구, 커뮤니케이션 도구가 각각 분리되어 비효율적
- **자막 출력 한계**: 대부분 OBS나 단순 텍스트 오버레이를 사용하며, 전문 자막 도구가 부재
- **협업 환경 부족**: 속기사 간 실시간 상태 공유, 교대 알림, 대기 시스템이 없음

### 대상 사용자
- **1차**: 전문 속기사 (행사/방송/회의 실시간 자막 담당)
- **2차**: 이벤트 운영자, 방송 PD (자막 출력 관리)
- **3차**: 접근성이 필요한 청각장애인 커뮤니티

## 2. 제품 목표

> 속기사가 실시간으로 협업하며 자막을 입력·송출·교대할 수 있는 올인원 자막 스튜디오

### 핵심 가치
1. **원스톱 작업 환경**: 입력 → 스타일링 → 송출을 하나의 앱에서
2. **끊김 없는 교대**: 속기사 간 실시간 교대 시 자막 끊김 제로
3. **전문 자막 품질**: 방송급 자막 스타일링 (폰트, 색상, 애니메이션)
4. **유연한 출력**: 듀얼 모니터, OBS 브라우저 소스, 웹 URL 공유

## 3. 기능 명세

### MVP (Sprint 1~3) — 현재 완료

| 기능 | 설명 | 상태 |
|------|------|------|
| 속기사 입력 패널 | 최대 4명의 독립 입력창, Enter로 자막 즉시 갱신 | DONE |
| 송출 전환 | 버튼 + 단축키(F1~F4)로 송출 담당자 실시간 전환 | DONE |
| 자막 출력 창 | 듀얼 모니터 투명 오버레이, 자유 크기 조절, 드래그 이동 | DONE |
| 자막 스타일 설정 | 폰트/크기/색상/배경/정렬/위치/줄간격/자간 실시간 조절 | DONE |
| 스타일 프리셋 | 기본/방송/강연/영화/고대비 5가지 프리셋 | DONE |
| 사용자 인증 | 게스트 로그인 (닉네임 → JWT), 카카오 OAuth 준비 | DONE |
| 설정 영속성 | 스타일/속기사 설정 자동 저장 및 앱 재시작 시 복원 | DONE |

### Phase 2 (Sprint 4) — 세션 관리

| 기능 | 설명 |
|------|------|
| 세션 생성/참가 | 작업 공간 개념 도입, 초대 코드로 참가 |
| 역할 자동 배정 | 참가 순서대로 operator → standby 자동 배정 |
| 자막 로그 저장 | 세션별 자막 이력 DB 저장 + 다운로드 |

### Phase 3 (Sprint 5~6) — 실시간 협업

| 기능 | 설명 |
|------|------|
| WebSocket 실시간 통신 | 여러 PC에서 같은 세션에 접속, 자막 실시간 동기화 |
| 교대 시스템 | standby → operator 교대 요청/승인, 자동 교대 타이머 |
| 대기 영역 | standby 속기사 전용 모니터링 + 연습 입력 화면 |

### Phase 4 (Sprint 7~8) — 웹 전환 + 안정화

| 기능 | 설명 |
|------|------|
| 웹 클라이언트 | 브라우저에서 동일 기능 사용 (Electron 불필요) |
| OBS 호환 | 자막 출력 URL을 OBS 브라우저 소스로 사용 |
| E2E 테스트 | 전체 플로우 자동화 테스트 |
| 배포 | 웹 서비스 배포 + Electron 인스톨러 |

## 4. 기존 솔루션과의 차별점

| 비교 항목 | 기존 도구 (OBS, 메모장 등) | Realtime Caption Studio |
|-----------|--------------------------|------------------------|
| 입력과 출력 | 별도 도구 조합 필요 | 원스톱 통합 |
| 속기사 교대 | 수동 (구두/채팅 소통) | 시스템화된 교대 플로우 |
| 자막 스타일 | 제한적 (OBS 텍스트 소스) | 전문 스타일링 + 프리셋 |
| 다중 속기사 | 지원 안 함 | 최대 4명 실시간 협업 |
| 대기 시스템 | 없음 | 대기/교대 자동화 |
| 출력 방식 | 특정 도구 종속 | 듀얼 모니터 / 웹 URL / OBS |

## 5. 기술적 제약사항

- MVP는 단일 PC + 듀얼 모니터 환경 (Phase 3부터 다중 PC)
- 자막 갱신 지연 50ms 이하 목표
- SQLite(개발) → PostgreSQL(프로덕션) 마이그레이션 필요
- 카카오 OAuth: 사내 네트워크 SSL 이슈로 보류 중

## 6. 성공 지표

- 2명의 속기사가 30분간 교대 없이 자막 입력 가능
- 자막 스타일 변경이 200ms 이내에 출력 화면에 반영
- 송출 전환 시 자막 끊김 0건
- 세션당 평균 자막 로그 1000줄 이상 저장 가능

## 7. 시장 분석

### 국내 속기 시장
- 국내 전문 속기사 약 2,000~3,000명 (법원, 국회, 방송, 프리랜서)
- 실시간 자막 서비스 시장 연간 약 500억원 규모 (2025년 기준)
- 성장 동인: 장애인차별금지법 강화, 온라인 행사 증가, ESG 경영 확대

### 글로벌 자막/캡션 시장
- 2025년 $4.2B → 2030년 $8.7B 예상 (CAGR 15.7%)
- 주요 성장 영역: 라이브 이벤트, 교육, 기업 회의

### 경쟁 환경

| 솔루션 | 실시간 입력 | 다중 속기사 | 교대 시스템 | 스타일링 | 협업 |
|--------|-----------|-----------|-----------|---------|------|
| OBS 텍스트 | 수동 | X | X | 제한적 | X |
| Vrew | 후편집 | X | X | O | X |
| Google Docs | O | O | X | X | O |
| **Caption Studio** | **O** | **O (최대 4명)** | **O (자동)** | **O (5 프리셋)** | **O (WebSocket)** |

### 핵심 차별화
1. **실시간 + 협업 + 교대의 삼위일체**: 기존 도구는 이 세 가지를 동시에 제공하지 않음
2. **무중단 교대**: 50ms 이내 역할 전환으로 자막 끊김 제로
3. **전문 속기사 특화**: 일반 자막 도구가 아닌 속기 현장의 워크플로우에 최적화

## 8. 기술 스택 선택 근거

| 기술 | 선택 이유 |
|------|-----------|
| Electron | 듀얼 모니터 투명 오버레이, 시스템 폰트 접근, 데스크탑 전용 기능 |
| React + TypeScript | 컴포넌트 재사용성, 타입 안정성, 웹 전환 시 코드 재사용 |
| Zustand | 경량 상태 관리, Redux 대비 보일러플레이트 90% 감소 |
| Socket.io | 자동 재연결, room 추상화, 폴백 지원, 실시간 양방향 통신 |
| Prisma + SQLite | 제로 설정 DB, 타입 안전한 ORM, PostgreSQL 마이그레이션 용이 |
| Tailwind CSS | 유틸리티 우선, 커스텀 디자인 토큰, 빠른 프로토타이핑 |
