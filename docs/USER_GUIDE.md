## 1. 설치

### 시스템 요구사항
- Windows 10 이상 (64bit)
- 듀얼 모니터 권장 (자막 출력용)

### 설치 방법

1. `release/` 폴더에서 최신 NSIS 인스톨러 파일(`.exe`)을 다운로드합니다.
2. 인스톨러를 실행하고 설치 경로를 선택합니다. (기본값 권장)
3. 설치가 완료되면 바탕 화면 또는 시작 메뉴에서 **Realtime Caption Studio**를 실행합니다.

### 서버 실행 (세션 모드 사용 시)

세션 모드로 협업하려면 백엔드 서버가 실행 중이어야 합니다.

```bash
cd server
cp .env.example .env    # 환경 변수 설정
npm install
npm run dev             # http://localhost:3000에서 서버 시작
```

## 2. 로그인

앱을 실행하면 로그인 화면이 나타납니다.

### 게스트 로그인
1. **닉네임** 입력란에 사용할 이름을 입력합니다. (2자 이상, 최대 20자)
2. **시작하기** 버튼을 클릭하거나 `Enter` 키를 누릅니다.
3. 서버에 접속되면 자동으로 세션 로비로 이동합니다.

### SNS 로그인
- **카카오 로그인**: 현재 준비 중입니다. (SSL 인증서 이슈로 보류)
- **네이버 로그인**: 향후 추가 예정입니다.

> 서버에 연결할 수 없는 경우 "서버 연결에 실패했습니다" 메시지가 표시됩니다. 서버가 실행 중인지 확인해 주세요.

## 3. 세션 생성 / 참가

로그인 후 **세션 로비** 화면에서 세션을 생성하거나 기존 세션에 참가할 수 있습니다.

### 세션 생성
1. 우측 상단의 **새 세션** 버튼을 클릭합니다.
2. 생성 폼이 나타나면 다음 정보를 입력합니다:
   - **세션 이름**: 작업 내용을 식별할 수 있는 이름 (예: "3월 15일 세미나 자막")
   - **최대 송출자 수**: 1~4명 중 선택
3. **세션 생성** 버튼을 클릭하면 세션이 만들어지고 자동으로 입장합니다.
4. 세션을 생성한 사용자가 첫 번째 operator(송출 담당자)가 됩니다.

### 세션 참가
1. 세션 로비에서 활성 세션 목록을 확인합니다.
2. 참가하고자 하는 세션의 **참가** 버튼을 클릭합니다.
3. 이미 operator가 있는 세션에 참가하면 standby(대기) 역할로 입장합니다.

### 세션 목록 새로고침
- **새로고침** 버튼을 클릭하면 최신 세션 목록을 다시 불러옵니다.

### 세션 나가기 / 종료
- **일반 참가자**: 헤더 우측의 **나가기** 버튼으로 세션에서 퇴장합니다.
- **세션 생성자**: **세션 종료** 버튼으로 세션을 완전히 종료합니다. 모든 참가자에게 종료가 알림됩니다.

## 4. 자막 송출

Realtime Caption Studio는 두 가지 모드로 자막을 송출합니다.

### 세션 모드 (서버 연결)

세션에 입장하면 WebSocket을 통해 실시간 협업이 가능합니다.

- **Operator(송출자)**: 텍스트 입력 패널에서 자막을 입력합니다. `Enter`를 누르면 자막이 즉시 송출됩니다.
- **Standby(대기자)**: 현재 operator가 송출 중인 자막을 실시간으로 모니터링합니다. 별도의 연습 입력 영역에서 타이핑 연습이 가능하며, 이 내용은 송출되지 않습니다.
- 송출된 자막은 WebSocket을 통해 모든 세션 참가자에게 전달됩니다.
- 자막 창이 열려 있으면 자막이 실시간으로 표시됩니다.

### 로컬 모드 (서버 미연결)

세션 없이 단독으로 사용하는 모드입니다.

- 속기사 A~D 패널 중 활성화된(LIVE) 패널에서 입력한 자막이 자막 창으로 송출됩니다.
- `F1`~`F4` 단축키 또는 ControlBar의 버튼으로 송출 담당 속기사를 전환합니다.
- Electron IPC를 통해 자막 창에 즉시 반영됩니다.

## 5. 교대 (Operator Handoff)

세션 모드에서 송출 담당자를 교대하는 방법은 세 가지입니다.

### Operator가 넘기기
1. 현재 operator는 ControlBar에서 standby 멤버 목록을 확인합니다.
2. 넘기고 싶은 멤버의 **-> [이름]** 버튼을 클릭합니다.
3. 즉시 해당 멤버가 새로운 operator가 되고, 기존 operator는 standby로 전환됩니다.

### Standby가 교대 요청
1. Standby 사용자는 화면 하단의 **송출 교대 요청** 버튼을 클릭합니다.
2. 또는 ControlBar의 **송출 교대 요청** 버튼을 사용합니다.
3. 요청이 승인되면 즉시 역할이 교체됩니다.

### 자동 승격
- Operator가 연결이 끊기거나(이탈) 세션을 나가면, 대기 중인 첫 번째 standby가 자동으로 operator로 승격됩니다.

> 역할이 변경되면 모든 참가자의 UI가 자동으로 업데이트됩니다. Operator는 입력 패널이, Standby는 모니터링+연습 패널이 표시됩니다.

## 6. 스타일 설정

자막의 시각적 스타일을 세밀하게 조정할 수 있습니다.

### 스타일 패널 열기
- ControlBar의 **스타일** 버튼을 클릭하면 우측에 스타일 설정 패널이 슬라이드 인됩니다.
- 패널 좌측 경계를 드래그하여 너비를 조절할 수 있습니다. (240px ~ 600px)

### 조정 가능한 항목

| 항목 | 설명 | 범위 |
|------|------|------|
| **프리셋** | 미리 정의된 스타일 세트를 한 번에 적용 | 기본, 방송용, 강연용, 고대비, 미니멀 |
| **폰트** | 글꼴 선택 | Noto Sans KR, 맑은 고딕, 나눔고딕, 나눔명조, 바탕, 굴림, 돋움, Arial, Helvetica, Verdana |
| **글자 크기** | 자막 텍스트 크기 | 24px ~ 96px |
| **글자색** | 텍스트 색상 | 색상 선택기 (HEX) |
| **배경색** | 자막 배경 색상 | 색상 선택기 (HEX) |
| **배경 투명도** | 배경의 불투명도 | 0% ~ 100% |
| **텍스트 정렬** | 텍스트 수평 정렬 | 좌측 / 가운데 / 우측 |
| **자막 위치** | 자막의 수직 위치 | 상단 / 중앙 / 하단 |
| **줄 간격** | 텍스트 줄 사이 간격 | 1.0 ~ 2.0 |
| **자간** | 글자 사이 간격 | -2px ~ 5px |

### 프리셋 상세

| 프리셋 | 용도 | 특징 |
|--------|------|------|
| **기본** | 일반 자막 | Noto Sans KR 48px, 흰색 텍스트, 검정 배경 70% |
| **방송용** | 방송 자막 | 맑은 고딕 56px, 검정 배경 85%, 자간 +1px |
| **강연용** | 강연/세미나 | Noto Sans KR 40px, 노란색 텍스트, 좌측 정렬 |
| **고대비** | 시인성 중시 | 맑은 고딕 52px, 파란 배경 90% |
| **미니멀** | 최소 간섭 | Noto Sans KR 36px, 배경 40% |

> 모든 스타일 변경은 자막 창에 실시간으로 반영됩니다.

## 7. 내보내기

세션의 자막 기록을 파일로 저장할 수 있습니다.

### 내보내기 방법
1. 세션 작업 화면의 헤더 우측에서 **TXT** 또는 **JSON** 버튼을 클릭합니다.
2. 파일이 자동으로 다운로드됩니다.

### 파일 형식

- **TXT**: 텍스트 형식으로 자막 로그를 포함합니다. 세션 정보(세션 이름, 생성자, 시작/종료 시간)가 상단에 기록됩니다.
- **JSON**: 구조화된 JSON 형식으로 세션 메타데이터와 전체 자막 로그가 포함됩니다. 프로그래밍 방식의 후처리에 적합합니다.

> 내보내기 파일명은 `session-[세션ID].[형식]`으로 저장됩니다.

## 8. 단축키

| 단축키 | 동작 | 비고 |
|--------|------|------|
| `F1` | 속기사 A로 전환 | 로컬 모드 전용 |
| `F2` | 속기사 B로 전환 | 로컬 모드 전용 |
| `F3` | 속기사 C로 전환 | 로컬 모드 전용 |
| `F4` | 속기사 D로 전환 | 로컬 모드 전용 |
| `Enter` | 자막 송출 / 로그인 확인 | 맥락에 따라 동작 |

> 세션 모드에서는 `F1`~`F4` 단축키가 비활성화됩니다. 세션 모드에서의 송출 전환은 교대 시스템을 사용합니다.

## 9. 자막 창

자막 창은 별도 윈도우로 열리며, 주로 보조 모니터(듀얼 모니터)에 표시하여 사용합니다.

### 자막 창 열기/닫기
- ControlBar의 **자막 창 열기** 버튼을 클릭하면 자막 출력 윈도우가 열립니다.
- 자막 창이 열린 상태에서 같은 위치의 **자막 창 닫기** 버튼을 클릭하면 닫힙니다.
- 자막 창이 외부에서 닫힌 경우(X 버튼 등) 메인 화면의 버튼 상태가 자동으로 동기화됩니다.

### 자막 창 특징
- **투명 배경**: 자막 창은 투명 오버레이 방식으로 동작하여 다른 콘텐츠 위에 자막을 겹쳐 표시할 수 있습니다.
- **실시간 반영**: 스타일 설정 변경이 즉시 자막 창에 적용됩니다.
- **독립 윈도우**: 메인 작업 화면과 별도 윈도우로 동작하므로, 보조 모니터로 이동시켜 사용할 수 있습니다.

### 권장 사용법
1. 메인 모니터: 속기사 작업 화면 (자막 입력, 교대, 스타일 설정)
2. 보조 모니터: 자막 창 (청중/시청자에게 보여지는 화면)

## 용어 정리

| 용어 | 설명 |
|------|------|
| **Operator** | 현재 자막을 송출하고 있는 속기사. 입력한 내용이 자막 창에 실시간으로 표시됩니다. |
| **Standby** | 대기 중인 속기사. 현재 자막을 모니터링하며 교대를 요청할 수 있습니다. |
| **Session** | 속기사들이 협업하는 작업 공간. 세션 내에서 자막 송출, 교대, 내보내기가 이루어집니다. |
| **로컬 모드** | 서버 없이 단일 PC에서 동작하는 모드. 속기사 A~D를 F1~F4로 전환합니다. |
| **세션 모드** | 서버에 연결하여 여러 속기사가 협업하는 모드. WebSocket으로 실시간 통신합니다. |

## 문제 해결

### 서버 연결 실패
- 백엔드 서버가 `http://localhost:3000`에서 실행 중인지 확인합니다.
- `server/.env` 파일이 올바르게 설정되어 있는지 확인합니다.

### 자막 창이 열리지 않는 경우
- ControlBar의 자막 창 열기 버튼을 다시 클릭합니다.
- 앱을 재시작한 후 다시 시도합니다.

### 자막 지연이 큰 경우
- 하단 상태바에서 지연 시간(ms)을 확인합니다. 200ms 이상이면 노란색으로 표시됩니다.
- 네트워크 상태를 확인하거나 서버를 재시작합니다.
