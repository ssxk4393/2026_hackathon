## 스킵 사유

### 의사결정 배경
Sprint 1~6에서 Electron 데스크탑 앱의 핵심 기능(자막 입출력, 세션 협업, WebSocket 실시간 통신, 교대 시스템)이 모두 구현 완료된 상태에서, Sprint 7(웹 클라이언트)과 Sprint 8(안정화) 중 우선순위를 결정해야 했다.

### 기술적 판단
1. **안정화 우선**: 웹 클라이언트보다 기존 코드의 테스트/에러 핸들링/성능 최적화가 제품 품질에 더 큰 영향
2. **코드 재사용 설계 완료**: React 컴포넌트와 Zustand 스토어는 이미 Electron 종속성 없이 설계되어, 웹 전환 시 최소 수정으로 재사용 가능
3. **Socket.io 호환성**: 클라이언트 Socket.io 코드는 브라우저에서도 동일하게 동작하므로 별도 작업 불필요
4. **IPC 추상화**: `window.electronAPI` 호출부만 분리하면 웹 환경 대응 가능 (이미 서비스 레이어로 분리됨)

### 리스크 평가
- 웹 클라이언트 구축 시 Electron 전용 코드(IPC, BrowserWindow)와 웹 호환 코드의 분리 작업 필요
- OBS 브라우저 소스용 자막 뷰어 페이지는 서버 사이드 렌더링 없이 정적 HTML + Socket.io로 구현 가능
- 배포 인프라(Vercel/AWS) 구성이 추가로 필요

### 결론
Sprint 8(안정화 + 릴리스)을 우선 진행하여 코드 품질과 테스트 커버리지를 확보한 후, Phase 2에서 웹 클라이언트를 구축하는 것이 합리적이라 판단.
